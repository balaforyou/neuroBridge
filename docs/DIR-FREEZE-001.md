# DIR-FREEZE-001

## Directions Activity Family Architecture Freeze

Status: Frozen

## 1. Vision

The Directions activity family should help the learner understand and respond to spatial direction instructions using a scaffolded progression.

Core targets:
- Direction recognition
- Spatial orientation
- Instruction following
- Visual attention
- Cognitive flexibility
- Working memory

---

## 2. Learner Context

The activity is designed specifically for Adarsh-style learning progression:
- **Simple visual directions first**: Prioritize basic orientation over language density.
- **Minimal language load**: Keep prompts short, calm, and focused.
- **Clear icon/arrow supports**: Use visual cues to reduce semantic retrieval load.
- **Scaffolded fading of supports**: Gradually remove visual cues as accuracy stabilizes.
- **No time pressure initially**: Ensure the learner can process and respond at their own pace without stress.

---

## 3. Learning Philosophy

NeuroBridge activities follow a structured learning and fading progression:

```text
Learn -> Practice -> Memory -> Mastery -> Fluency
```

Progression is based on demonstrated, stable mastery over time rather than simple session completion.

---

## 4. Direction Concepts

Supported initial directions:
- Up
- Down
- Left
- Right

Supported later (deferred after V1):
- Forward
- Backward
- Turn left
- Turn right
- Diagonal directions
- Multi-step directions

---

## 5. V1 Activity Model

Start with arrow-in-box direction recognition.

Learner sees:
- A rectangle or grid area containing directional options.
- An arrow or target object.
- A spoken and written prompt, such as:
  * "Tap up"
  * "Tap left"

Learner responds by selecting the matching direction button.

---

## 6. Scaffold Model

Visual scaffold fading levels:
- **Full visual arrow support**: Clear arrows matching the target direction.
- **Highlighted border support**: Gentle border cues highlighting target choices.
- **Reduced visual cue**: Muted or scaled-down arrows.
- **Prompt-only direction**: Written/spoken instruction with no visual arrow indicators.
- **Memory / delayed response**: A delay introduced between prompt presentation and selection.

Scaffold Rule:
* Cues and support levels fade only after consistent learner success.

---

## 7. Level Progression

Level 1:
- Single direction prompt.
- Four large choice cards.
- Arrow/icon visible.
- No timer.

Level 2:
- Border-highlight scaffold.
- Direction word + visual cue.

Level 3:
- Direction word only.
- No arrow cue.

Level 4:
- Two-step direction sequence.
  * *Example*: "Up then Left"

Level 5:
- Mixed directions / cognitive flexibility.
  * *Example*: Follow prompt, then select the opposite direction.

Fluency:
- A timer is introduced only after stable mastery and natural reaction time profiles are established.

---

## 8. Feedback

Leverage the standard SIRAASH Feedback Engine:
- **Correct Selection**: Standard click/success feedback card and sound.
- **Wrong Selection**: Gentle orange pulse. Avoid red error indicators, cross marks, or harsh buzzers.
- **Perfect Round**: Zero-error success sound.
- **Level Advancement**: Celebration effects (e.g. level-up sound and banner).

---

## 9. Session Structure

V1 Session bounds:
- Small number of trials per session: 8 to 12 trials.
- Completion summary screen is required (showing accuracy, corrects, hints, and time).
- Try Again / Play Again and Return Home actions must remain visible.

---

## 10. Analytics

Track and store:
- Accuracy
- Correct selections
- Incorrect selections
- Prompt type
- Direction type
- Scaffold level
- Response time
- Completion status
- Session duration

---

## 11. Skill Mapping Draft

Domains:
- Spatial Processing
- Executive Function
- Attention
- Working Memory

Mode / level evidence mapping examples:
- **Single direction recognition**: Spatial Orientation, Visual Attention.
- **Direction word only**: Receptive Language, Symbol-to-action mapping.
- **Multi-step direction**: Working Memory, Sequencing.
- **Opposite direction**: Cognitive Flexibility, Inhibitory Control.

---

## 12. Practice Lab

Practice Lab is deferred:
- Parents can choose direction options in a playground, but this must not contaminate or influence official learning path metrics.

---

## 13. Supported V1

- Up / Down / Left / Right concepts.
- Single-step direction recognition.
- Visual arrow support.
- Gentle non-punitive feedback.
- Persistent completion summary screen.
- Basic analytics aggregation.

---

## 14. Not Supported V1

- Multi-step directions.
- Opposite-direction rules.
- Diagonal directions.
- Timers or speed pressure.
- Adaptive fluency.
- Parent Practice Lab controls.
- Complex maps or spatial mazes.
- Avatar or character movement.
- Voice recognition.

---

## 15. Potential Confusion Areas

- **Direction recognition is not navigation**: In V1, the task is selecting the abstract direction, not driving an avatar through a map.
- **Arrow support is not word-only**: Arrow support is a low-cognitive-load scaffold; word-only requires receptive symbol mapping.
- **Practice mode is not learning path**: Practice sandbox play does not update official learning progression.
- **Timer is not mastery**: Mastery requires stable accuracy; speed is a secondary fluency target.
- **Multi-step instruction is not V1**: Compound sequences belong in Level 4+.

---

## 16. Definition of Success

V1 is successful when the learner can complete simple Up / Down / Left / Right direction recognition trials with clear feedback, telemetry capture, and a persistent completion summary.
